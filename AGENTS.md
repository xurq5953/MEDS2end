# AGENTS.md

## Project purpose

This repository starts from the MEDS reference-style implementation and is being modified into `MEDS2endGen`.

The current target is not a generic refactor. It is a cryptographic algorithm modification with a precise mathematical specification. Preserve existing MEDS infrastructure where possible, but replace the original MEDS public-key compression and two-matrix action with the `MEDS2endGen` design described in `meds2end.tex`.

## Algorithm specification priority

The file `docs/meds2end.tex` is the highest-priority specification for the MEDS2endGen algorithm.

When `docs/meds2end.tex`, the original MEDS document, and the current C reference code disagree, follow `docs/meds2end.tex`.

Before modifying KeyGen, Sign, Verify, key formats, signature formats, or the group action implementation, read `docs/meds2end.tex`.

## Mathematical target: MEDS2endGen

Use the special setting

```text
m = n = k
```

The current parameter set is expected to use `MEDS_m = MEDS_n = MEDS_k = 14`, but code should use macros rather than hard-coded `14` unless a test explicitly needs a concrete value.

A public generator matrix has shape

```text
G in F_q^{n x n^2}
```

The row-to-matrix interpretation is:

```text
(G_1, ..., G_n) = RowsToMatrices(G)
```

where each `G_i` is an `n x n` matrix over `F_q`. In the C implementation, this normally corresponds to row-major storage of the `n` row blocks of length `n*n`.

The modified group action is

```text
Phi_{A,B,C}(G) = MatricesToRows(G'_1, ..., G'_n)
```

with

```text
G'_j = sum_i C[j,i] * A * G_i * B
```

using zero-based C loops:

```c
for (j = 0; j < MEDS_n; j++) {
    Gprime_j = 0;
    for (i = 0; i < MEDS_n; i++) {
        Gprime_j += C[j * MEDS_n + i] * A * G_i * B;
    }
}
```

Do not reverse the index order of `C`. The output block `j` uses row `j` of `C`.

When `C` is the identity, `Phi_{A,B,C}` reduces to applying the original left/right matrix action independently to each generator matrix.

## Critical algorithm rules

### Public-key compression must be removed

Do not use the original MEDS public-key compression path in final MEDS2endGen code.

Remove or bypass:

- `Solve` / `solve()` for deriving `A` and `B` from fixed public codewords.
- The secret `T_i` basis-change path used only for MEDS public-key compression.
- `CompressG` / `DecompressG` semantics that store only special parts of `G_i`.
- Special public-key reconstruction that hard-codes the first matrix as identity and the second as an upper-shift matrix.

Ordinary bitstream packing of field elements with `GFq_bits` is still allowed. That is serialization, not the MEDS public-key compression technique.

### Do not apply systematic form to Phi outputs

In final MEDS2endGen code, do not call `pmod_mat_syst_ct()` or any `SF` operation on:

- public matrices `G_a = Phi_{A_a,B_a,C_a}(G_0)`, for `a > 0`;
- commitment matrices `G_tilde_j = Phi_{A_tilde_j,B_tilde_j,C_tilde_j}(G_0)`;
- verifier-reconstructed matrices `G_hat_j`.

`G_0` may still be generated by the existing `rnd_sys_mat` / `ExpandSystMat` logic from `sigma_G0`, unless the task explicitly changes this.

The reason is that `C` acts on the generator-index space. Applying a canonical systematic form would remove or hide this basis information and make `C` ineffective.

### Hash the full encoded matrix

In final MEDS2endGen signing and verification, hash the full encoding of each commitment/reconstructed matrix:

```text
H( Enc(G_0_tilde) || ... || Enc(G_{t-1}_tilde) || message )
```

Do not hash only the systematic tail `[; k, mn - 1]` after final migration, because final matrices are not systematic.

For the current `n=14`, full `G` encoding length is

```text
MEDS_G_BYTES = CEILING(MEDS_k * MEDS_m * MEDS_n * GFq_bits, 8)
             = CEILING(14 * 14 * 14 * 12, 8)
             = 4116 bytes
```

Use a macro rather than a literal value.

### Key generation target

For each public key index `a = 1 .. MEDS_s - 1`:

1. Generate `A_a`, `B_a`, and `C_a` as invertible `n x n` matrices.
2. Store `A_a^{-1}`, `B_a^{-1}`, and `C_a^{-1}` in the secret key.
3. Compute

```text
G_a = Phi_{A_a,B_a,C_a}(G_0)
```

4. Store the full encoded `G_a` in the public key, unless the task explicitly says to store `G_0` and all `G_a` fully. Keeping `G_0` represented by `sigma_G0` is acceptable because `meds2end.tex` allows public matrices to be stored explicitly or generated from public seeds.

### Signing target

For each round `j = 0 .. MEDS_t - 1`:

1. Generate temporary invertible matrices

```text
A_tilde_j, B_tilde_j, C_tilde_j in GL_n(q)
```

2. Compute

```text
G_tilde_j = Phi_{A_tilde_j,B_tilde_j,C_tilde_j}(G_0)
```

3. Absorb `Enc(G_tilde_j)` into the Fiat-Shamir hash.

For each nonzero challenge `h[j] > 0`, the response must contain three matrices:

```text
mu_j  = A_tilde_j * A_inv[h[j]]
nu_j  = B_inv[h[j]] * B_tilde_j
eta_j = C_tilde_j * C_inv[h[j]]
```

The multiplication order is part of correctness. Do not change it.

### Verification target

For each round `j`:

- If `h[j] == 0`, recover the seed and regenerate

```text
A_hat_j, B_hat_j, C_hat_j
```

then compute

```text
G_hat_j = Phi_{A_hat_j,B_hat_j,C_hat_j}(G_0)
```

- If `h[j] > 0`, read `mu_j`, `nu_j`, and `eta_j` from the response stream, check all three are invertible, and compute

```text
G_hat_j = Phi_{mu_j,nu_j,eta_j}(G[h[j]])
```

Then hash the full `Enc(G_hat_j)` and compare the recomputed digest with the digest in the signature.

## Expected size formulas for current MEDS-9923-derived parameters

Assuming the repository keeps `G_0` represented by `sigma_G0` and stores full `G_a` for `a = 1 .. s-1`, define:

```c
#define MEDS_MAT_BYTES CEILING((MEDS_n * MEDS_n) * GFq_bits, 8)
#define MEDS_G_BYTES   CEILING((MEDS_k * MEDS_m * MEDS_n) * GFq_bits, 8)
```

For the current expected parameter values:

```text
q = 4093
GFq_bits = 12
MEDS_m = MEDS_n = MEDS_k = 14
MEDS_s = 4
MEDS_t = 1152
MEDS_w = 14
MEDS_MAT_BYTES = 294
MEDS_G_BYTES = 4116
```

Expected byte sizes after final MEDS2endGen migration:

```text
MEDS_PK_BYTES  = MEDS_pub_seed_bytes + (MEDS_s - 1) * MEDS_G_BYTES
               = 32 + 3 * 4116
               = 12380

MEDS_SK_BYTES  = MEDS_sec_seed_bytes + MEDS_pub_seed_bytes
                 + 3 * (MEDS_s - 1) * MEDS_MAT_BYTES
               = 32 + 32 + 3 * 3 * 294
               = 2710

MEDS_SIG_BYTES = old_nonresponse_part + MEDS_w * 3 * MEDS_MAT_BYTES
               = 1664 + 14 * 3 * 294
               = 14012
```

If the task changes whether `G_0` is stored as a seed or as a full matrix, recompute these sizes and update both `params.h` and `api.h` together.

## Implementation guidance

### Add a new Phi function

Prefer adding a dedicated function, for example:

```c
void phi(pmod_mat_t *G_out,
         const pmod_mat_t *A,
         const pmod_mat_t *B,
         const pmod_mat_t *C,
         const pmod_mat_t *G_in);
```

Do not silently change the semantics of the existing `pi()` until all call sites are audited. It is safer to keep `pi()` for the original two-matrix action and call `phi()` for MEDS2endGen.

The `phi()` function must be tested on small hand-checkable cases before being used in signing and verification. At minimum, test:

- `C = I`, where `phi(A,B,I,G)` equals applying the old `pi(A,B,G)` blockwise.
- `A = I`, `B = I`, where `phi(I,I,C,G)` only mixes the generator-index blocks by `C`.
- A composition/correctness case matching

```text
Phi_{mu,nu,eta}( Phi_{A,B,C}(G) ) = Phi_{A_tilde,B_tilde,C_tilde}(G)
```

when

```text
mu  = A_tilde * A^{-1}
nu  = B^{-1} * B_tilde
eta = C_tilde * C^{-1}
```

### Serialization guidance

Use the existing bitstream packing style. Always call `bs_finalize()` at the same matrix boundaries used by the corresponding reader.

Public key after final migration:

```text
sigma_G0 || EncFull(G_1) || ... || EncFull(G_{s-1})
```

Secret key after final migration:

```text
delta || sigma_G0 || Enc(A_1^{-1}) || ... || Enc(A_{s-1}^{-1})
                  || Enc(B_1^{-1}) || ... || Enc(B_{s-1}^{-1})
                  || Enc(C_1^{-1}) || ... || Enc(C_{s-1}^{-1})
```

Signature response section after final migration:

```text
for each nonzero h[j]: Enc(mu_j) || Enc(nu_j) || Enc(eta_j)
```

The rest of the signature remains:

```text
responses || seed_tree_path || digest || salt || message
```

unless the task explicitly changes the signed-message layout.

### Matrix and field arithmetic

Reuse existing finite-field helpers and matrix multiplication helpers. Do not introduce new arithmetic that bypasses modular reduction.

Maintain row-major indexing consistently. Avoid changing global finite-field types such as `GFq_t` or `pmod_mat_t` unless explicitly requested.

## Done criteria for algorithm-change tasks

A task is not complete until all relevant items are true:

- The code builds without warnings introduced by the change.
- Key generation, signing, and verification succeed for multiple random messages.
- Verification fails when the public key, signature, digest, or a response matrix is tampered with.
- `params.h` and `api.h` byte-size constants agree with actual serialized lengths.
- Signing and verification hash exactly the same byte sequence for all reconstructed commitments.
- No final MEDS2endGen path still depends on MEDS public-key compression, `solve()`, or systematic-form normalization of `Phi` outputs.
- Any changed KAT files are regenerated intentionally and mentioned in the final summary.
